# 要件定義書（タスク管理アプリ）

## 背景

現代人は毎日、タスクをこなしています。

しかし、それぞれのタスクをどのように優先順位をつけて進めていくかわからなかったり、適切にアウトプットができずにいます。また、それぞれのタスクがどれくらい密度が高いのかわからずに進めていくために、タスクの負荷も考慮したタスク管理ができないという問題があります。

## 課題

- それぞれのタスクの優先順位が適切に管理されていない
- それぞれのタスクに対して、しっかりアウトプットができていない
- それぞれのタスクの密度（負荷）を考慮した上で、タスクを管理できていない

## 目的

- タスクを適切に管理（優先順位、アウトプット、負荷）する仕組みを作る
- あとからタスク（アウトプット、負荷）を振り返りができるように、検索や比較や再利用ができるようにする

## 機能要件（MVP版）

### タスク機能

- タスクを作成できる（日付、タイトル、それぞれの子タスク（優先度、密度、タスク名、継続時間）、1日の振り返り）
- 自分が作ったタスクを編集、削除できる（物理削除）
- 自分が作ったタスクはマイページで一覧表示できる
- 自分が作ったタスクはマイページの詳細で確認できる
- タスクを検索できる（タスクのタイトルとそれぞれの子タスクの部分一致）
- 自分が作った子タスクで、子タスク終了後にアウトプットができる

## 将来の拡張予定

- それぞれ子タスクにカテゴリーを付けられる（例　思考系、文章系、インプット系）また、それらのカテゴリーについては自分でカテゴリー自体を設定できる
- それぞれ子タスクのアウトプットについてテンプレートからアウトプットの形式を選択できる
- 他人のタスクのアウトプットも見れる状態にする
- タスクの進め方について、優先度によってレコメンドされる
- 自分が作ったタスクで予定のタスク（タスク数と合計時間）、完了したタスク（タスク数と合計時間（タスクの負荷ごとも（例　高負荷◯時間、中負荷◯時間、低負荷◯時間））を表示できるようにする
- これまでのタスクの振り返り（１ヶ月ごとや１週間ごと、負荷ごと、カテゴリーごとなど）ができるようにする
- それぞれの子タスクで、タイマー機能をつけて時間がたったらアウトプットできるようにする

## 非機能要件（MVP）

- 認証：Googleログイン/ログアウト/初回登録（OAuth2）
- データベース：Postgres、正規化されたテーブル構造
- 検索：（タスクのタイトルとそれぞれの子タスクの部分一致）
- UI：PCブラウザ対応（スマホ最適化なし）
- パフォーマンス：軽量シングルユーザー規模を想定
- デプロイ：Cloud Run + Cloud Build（自動デプロイ）
- CI/CD：GitHub Actions
- 監視：Cloud Loggingのみ（最低限）

## 他ツールとの差別化

- それぞれのタスクについてアウトプットができ、あとで振り返りができる
- それぞれのタスクについて負荷も考慮にいれて、管理ができる
